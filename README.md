# WB + Ozon Analytics Dashboard

> **Live demo:** [b0gdaan.github.io/data_for_wb_ozon](https://b0gdaan.github.io/data_for_wb_ozon/)

Аналитический дашборд для продавцов на маркетплейсах **Wildberries** и **Ozon**.  
Автоматически забирает данные через официальные API, считает прибыль, остатки, воронку конверсии и показывает всё в едином интерфейсе.

**Стек:** Python · FastAPI · PostgreSQL · React · Nginx · Ubuntu

---

## Что это и зачем

Продавец на WB и Ozon видит данные только внутри каждого личного кабинета — они разрозненные, неудобные, без сводной аналитики. Этот инструмент:

- **Забирает данные** с обеих площадок раз в сутки через официальные API
- **Считает реальную прибыль** с учётом комиссий, логистики и себестоимости
- **Показывает всё в одном окне** — тёмный дашборд с 10 разделами
- **Отправляет Telegram-уведомления** когда товар заканчивается на складе

## Что показывает дашборд

| Раздел | Что внутри |
|--------|-----------|
| **Финансы** | Выручка и прибыль по дням, P&L-структура, графики WB vs Ozon |
| **Товары** | Рентабельность каждой позиции: ROI%, валовая прибыль, комиссия, логистика |
| **Склад** | Остатки на складах. Статус: СРОЧНО / ДОКУПИТЬ / OK — на основе скорости продаж. AI-рекомендация к закупу |
| **Карточки** | Статистика карточек: показы, CTR, конверсия в корзину и заказ |
| **WB vs Ozon** | Сравнение площадок: доля комиссии %, логистики %, чистая выручка |
| **Тренды** | Рост продаж неделя-к-неделе (WoW%) по каждой площадке |
| **Сравнение периодов** | Текущий месяц vs предыдущий — выручка, прибыль, заказы с дельтой % |
| **Топ / Флоп** | Лучшие и худшие позиции по прибыли + ABC-анализ |
| **Категории** | Разбивка выручки и прибыли по категориям, donut-chart |
| **Воронка** | Показы → Клики → Корзина → Заказ. CTR и CR по каждому товару |

Дополнительно: экспорт любого раздела в CSV, ручной запуск синхронизации прямо из дашборда.

## Технологии

- **Backend:** Python 3.10, FastAPI, psycopg2 — REST API с 20+ эндпоинтами
- **База данных:** PostgreSQL 14 — 5 таблиц, 9 аналитических вьюшек, оконные функции
- **Frontend:** React 18, SVG-charts, Manrope + JetBrains Mono — одна HTML-страница без сборки
- **ETL:** официальные API Wildberries (Statistics, Content, Analytics) и Ozon (Finance, FBS/FBO, Analytics)
- **Инфраструктура:** Ubuntu 22.04, Nginx (reverse proxy), systemd, cron
- **Demo:** GitHub Pages (статичная версия с mock-данными)

## Быстрый старт

### 1. Получить API-ключи

| Площадка | Где взять |
|----------|----------|
| **Wildberries** | seller.wildberries.ru → Настройки → Доступ к API → Создать токен (права: Статистика, Контент, Аналитика, Склад) |
| **Ozon** | seller.ozon.ru → Настройки → Seller API → Сгенерировать ключ (права: Analytics, Finance, Warehouses, Postings) |
| **Telegram** (опционально) | @BotFather → `/newbot` |

### 2. Клонировать и настроить

```bash
git clone https://github.com/b0gdaan/data_for_wb_ozon.git
cd data_for_wb_ozon
cp .env.example .env
nano .env   # вставить ключи API
```

### 3. Установить на сервер (Ubuntu 22.04)

```bash
sudo bash install.sh
```

Скрипт сам установит Python, PostgreSQL, Nginx, создаст базу, настроит systemd и cron.

### 4. Загрузить данные

```bash
# Первый импорт — последние 30 дней
python run_etl.py --days 30

# Только WB или только Ozon
python run_etl.py --wb
python run_etl.py --ozon
```

### 5. Открыть дашборд

```
http://ВАШ_IP_СЕРВЕРА
```

## Схема базы данных

```
products   — товары (source + sku = составной уникальный ключ)
sales      — продажи (выручка, комиссия, логистика, чистая выручка)
orders     — заказы
stocks     — остатки по складам и датам
card_stats — статистика карточек (показы, клики, добавления в корзину)
```

Один и тот же SKU может существовать независимо как WB-товар и как Ozon-товар — это намеренное решение для корректного сравнения площадок.

## Автоматизация

- **09:00** — Telegram-уведомление о товарах с критичными остатками
